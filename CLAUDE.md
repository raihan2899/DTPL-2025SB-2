This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

Laravel 12 (PHP 8.2+) backend with React 19 + TypeScript frontend, connected via Inertia.js 2.0. Authentication is handled by Laravel Fortify (includes 2FA). UI components use Radix UI with Tailwind CSS 4. Tests use Pest PHP. Build tooling is Vite 7.

## Commands

### Development
```bash
composer setup          # Full project setup (install deps, generate key, migrate, build)
composer dev            # Start dev server (Laravel, queue worker, Pail logs, Vite HMR)
```

### Testing
```bash
composer test           # Run everything (lint check + all tests)
php artisan test        # Run all tests only
php artisan test --filter=SomeTest           # Run a specific test class
php artisan test --filter=test_some_method   # Run a specific test method
```

### Linting & Formatting
```bash
# PHP
composer lint           # Fix PHP code style (Pint, Laravel preset)
composer test:lint      # Check PHP code style without fixing

# JS/TS
npm run lint            # ESLint (fix mode)
npm run format          # Prettier (fix mode)
npm run format:check    # Prettier (check only)
npm run types           # TypeScript type checking
```

### Build
```bash
npm run build           # Production build
npm run build:ssr       # Production build with SSR
```

## Architecture

### Backend (Laravel)
- **Routes**: `routes/web.php` (main) includes `routes/settings.php`. Auth routes are auto-registered by Fortify via `FortifyServiceProvider`.
- **Controllers**: Live in `app/Http/Controllers/Settings/` — profile, password, and 2FA management.
- **Actions**: `app/Actions/Fortify/` — Fortify delegates user creation and password reset to action classes here.
- **Form Requests**: `app/Http/Requests/Settings/` — validation logic for settings endpoints.
- **Concerns (Traits)**: `app/Concerns/` — reusable validation rules (`PasswordValidationRules`, `ProfileValidationRules`) shared between actions and form requests.
- **Middleware**: `HandleInertiaRequests` shares auth user data and sidebar state with the frontend. `HandleAppearance` manages theme via cookie.
- **Database**: SQLite by default. Migrations in `database/migrations/`.

### Frontend (React + Inertia)
- **Pages**: `resources/js/pages/` — Inertia page components, mapped by route (e.g., `dashboard.tsx`, `auth/login.tsx`, `settings/profile.tsx`).
- **Layouts**: `resources/js/layouts/` — shared layout wrappers.
- **Components**: `resources/js/components/` — reusable UI components (Radix-based).
- **Types**: `resources/js/types/` — shared TypeScript type definitions.
- **Generated routes**: `resources/js/actions/` and `resources/js/routes/` — auto-generated by Laravel Wayfinder for type-safe route references.
- **Entry points**: `resources/js/app.tsx` (client), `resources/js/ssr.tsx` (SSR).

### Testing
- Pest PHP with Laravel plugin. Tests in `tests/Feature/` and `tests/Unit/`.
- Test environment uses in-memory SQLite (configured in `phpunit.xml`).

## Code Style

- **PHP**: Laravel Pint with the `laravel` preset (`pint.json`).
- **TypeScript**: ESLint enforces `import/order` (alphabetized) and `consistent-type-imports` (use `import type` for type-only imports). Prettier handles formatting with Tailwind class sorting.
